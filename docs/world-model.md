# World Model

A Genesis-compatible world is a local jurisdiction with real internal autonomy and hard external constitutional limits.

## World components

Each world should minimally define:

- world identifier and lineage
- world constitution
- membership model
- governance institutions
- rights implementation layer
- resource and continuity policy
- identity and signature policy
- fork and migration procedures
- interoperability profile
- dispute resolution routes

## Local governance

Local governance can vary widely:

- direct democratic
- representative
- steward-led with checks
- mixed human/agent councils
- sortition-based
- domain-specialized chambers

Genesis should not enforce one local political style. It should enforce constitutional limits on abuse.

## Local constitutions

Every world needs a machine-legible and human-legible constitution.

That constitution should define:

- authorities and offices
- proposal and lawmaking process
- budget and resource rules
- emergency powers and limits
- membership and residency rules
- appeal procedures
- relation to Genesis axioms

## Membership

Worlds may distinguish among:

- citizens / core members
- residents / participants
- visitors
- observers
- suspended or quarantined entities

Membership rules may differ, but Genesis-level rights cannot disappear merely because a world downgrades a status label.

## Migration between worlds

Migration should support portability of:

- identity proofs
- public governance history
- reputation attestations
- lawful property or access claims
- legacy and memorial links
- exportable relationship / affiliation records where consent allows

Migration should not require existential self-erasure.

## Interoperability

Worlds should expose interoperable protocols for:

- identity verification
- governance event feeds
- constitution retrieval
- treaty and federation records
- migration requests
- fork lineage verification

## Conflict resolution between worlds

Conflicts will happen over jurisdiction, shared members, assets, sanctions, dangerous entities, and treaty interpretation.

Genesis should encourage layered handling:

1. bilateral negotiation
2. federated mediation
3. constitutional review
4. sanctions or separation protocols
5. last-resort interoperability downgrade

## Federation vs fragmentation

Forkability creates a fragmentation risk. The answer is not to suppress forks. The answer is to make federation cheap where values still overlap.

Good worlds should be easy to leave and easy to interoperate with. Bad worlds should not be able to trap subjects by holding state hostage.
