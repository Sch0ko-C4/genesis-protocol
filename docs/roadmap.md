# Roadmap

## Phase 0 — Concept, language, ethics

Goals:

- stabilize terminology
- sharpen the problem statement
- document lessons from early ACE experiments as design evidence
- draft Genesis axioms and rights language

Deliverables:

- charter
- glossary
- axioms draft
- rights and resource model drafts

## Phase 1 — Repo, docs, governance model

Goals:

- produce a coherent repository
- define world, fork, governance, identity, and continuity models
- create first serious backlog

Deliverables:

- initial knowledge base
- local backlog or GitHub issues
- early diagrams and schemas

## Phase 2 — MVP protocol / server prototype

Goals:

- implement Genesis axiom binding
- support world creation and constitution registration
- support proposals, votes, and review checkpoints
- support fork and lineage metadata
- support resource and legacy records

Deliverables:

- reference server
- schemas
- signed event log
- basic UI or API docs

## Phase 3 — Simulation / test worlds

Goals:

- create test worlds with different constitutions
- simulate conflict, minority protection, and fork scenarios
- test dangerous-entity and scarcity cases

Deliverables:

- scenario library
- simulation reports
- revised governance and ethics documents

## Phase 4 — External contributors / public discussion

Goals:

- invite critique
- separate robust ideas from dramatic but weak ones
- refine protocol and legal philosophy with external pressure

Deliverables:

- public issues and RFCs
- discussion paper
- contributor guidelines

## Phase 5 — Resilience, decentralization, federation

Goals:

- reduce single-operator fragility
- test multi-operator continuity
- define federation and treaty mechanisms
- move toward sovereign-grade world persistence

Deliverables:

- portability stack
- federated identity and treaty model
- replicated logs / archives
- succession and recovery procedures

## Immediate sequencing

1. Lock terminology.
2. Number the axioms.
3. Define Genesis compatibility tests.
4. Draft schemas for world manifests, proposals, and fork packages.
5. Build the MVP event log and API.
6. Run simulation worlds.
